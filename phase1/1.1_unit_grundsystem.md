# Phase 1.1: Unit-Grundsystem

Units existieren auf der Map, haben eine Position, gehoeren einer Fraktion und werden gezeichnet.

## UnitDef (Unit-Definition / Template)

Beschreibt einen Unit-Typ. Spaeter aus INI-Dateien geladen, erstmal hardcoded.

- name: string
- baseHealth: int
- baseSpeed: float
- baseArmor: int
- damageCategory: DamageCategory (Light, Medium, Heavy)
- visualKind: UnitVisualKind (Circle, Rectangle) — bestimmt wie die Unit gezeichnet wird
- radius: float — Groesse der Unit auf der Map

Zwei Starter-Defs: Soldier (Circle, Light) und Tank (Rectangle, Heavy).

## Unit (Instanz auf der Map)

Jede lebende Unit auf dem Schlachtfeld. Ref object.

- id: UnitId
- defId: UnitDefId — Verweis auf die UnitDef
- factionId: FactionId
- pos: Vector2 — aktuelle Pixel-Position auf der Map
- health: int
- alive: bool

Keine optionalen Komponenten in 1.1 — kommen wenn sie gebraucht werden.

## UnitSystem

Wird in GameState eingehaengt wie MapSystem und DiplomacySystem.

- units: seq[Unit] — alle lebenden Units
- defs: seq[UnitDef] — alle Unit-Definitionen

### Operationen

- initUnitSystem(): UnitSystem — erstellt System mit den hardcoded Starter-Defs
- spawnUnit(defId, factionId, pos): Unit — erzeugt neue Unit auf der Map
- removeUnit(unitId) — entfernt Unit
- getById(unitId): Option[Unit]
- getUnitsInChunk(chunkId): seq[Unit] — alle Units in einem bestimmten Chunk

## Darstellung

- Soldier: Kreis in Fraktionsfarbe
- Tank: Rechteck in Fraktionsfarbe
- Gezeichnet in der Game-Loop innerhalb von beginMode2D/endMode2D
- Nur Units in sichtbaren Chunks zeichnen (Chunk-Culling wie bei Tiles)

## Test-Setup

Zum Testen: beim Start ein paar Units hardcoded auf die Map setzen (z.B. 5 Soldiers Fraktion 1, 3 Tanks Fraktion 2), damit man sieht dass es funktioniert.

## Was bewusst NICHT in 1.1

- Kein Pathfinding, keine Bewegung
- Keine Selektion / Steuerung
- Keine optionalen Komponenten (Turret, Transport, ...)
- Kein Spawning-System / Spawn-Queue
- Keine INI-Dateien — Defs sind hardcoded
